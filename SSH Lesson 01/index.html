Learning Objectives

Master the implementation and troubleshooting of Secure Shell (SSH) for remote server management.

  • Review SSH syntax for successful endpoint connection.
  • Utilize SSH in standard Admin activities.
  • Troubleshoot SSH implementation failures.

Initial Connection

Attempt to establish a session with the NLC-SERVER device.

TARGET: NLC-SERVER
USER: NLC_User
PASS: Password1
IP: 192.168.2.220
COMMAND: ssh NLC_User@192.168.2.220

ROADBLOCKS: SSH service is likely OFF. Use Windows Services to enable the SSH service to allow functionality.

Administrative Actions

Document the commands necessary to modify users and retrieve sensitive data.

TASK_01: Create new user account
TASK_02: Add user to Local Administrators
TASK_03: Download C:\Credentials.txt
PERMISSIONS ISSUE: User creation and group modification will fail due to insufficient privileges.

DATA RETRIEVAL:
- Terminal: type C:\Credentials.txt
- SCP (External): scp NLC_User@192.168.2.220:\C:\Credentials.txt .

Wireshark Telemetry

Analyze packet captures to determine if SSH traffic remains secure during the previous steps.

PROTOCOL: SSHv2
CIPHER: AES-GCM
ANALYSIS_TOOL: Wireshark
VISIBILITY STATUS:
- Login Activity: NOTHING (Encrypted)
- User Creation: NOTHING (Encrypted)
- File Download: NOTHING (Encrypted)

DATA_GLOSSARY

SSH (Secure Shell) A cryptographic network protocol for operating network services securely over an unsecured network.
Port 22 The default network port assigned for SSH traffic.
SCP (Secure Copy) A means of securely transferring computer files between a local host and a remote host.
Encrypted The process of converting information or data into a code, especially to prevent unauthorized access.
